Система состоит из нескольких сервисов. В реальности с таким количеством пользователей без нагрузки на сервис проще сделать монолит. В учебной тренируемся проектировать.
Наличие нескольких сервисов позволит сделать систему более отказоустойчивой, компоненты продолжат работать при возникновении внештатных ситуаций.
Минус - увеличивается сложность разработки.

Сервисы:

1. сервис sso. Чтобы во всех сервисах не плодить какую-то кастомную авторизацию и унифицировать управление пользователями и правами доступа
2. сервис управления тасками. включает в себя все операции по управлению задачами для юзеров, воркеров, менеджеров. крон по обработке протухших задач и расчету стоимостей. 
тут же нотификации о счетах и лог + статистика счетов
3. сервис управления вакансиями. включает в себя все операции по управлению вакансиями для воркеров, менеджеров. тут же нотификации и всякое про статистику если надо
4. отдельный сервис про ставки. почти не связан с другими частями системы, можно быстренько удалить если придет налоговая
